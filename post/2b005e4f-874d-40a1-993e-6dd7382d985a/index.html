github desktop commit, push 원복시키기

타이틀 바텀... (날짜, 이름)


GitHub Desktop에서 푸시된 내용을 원복(되돌리기)하는 방법은 reset이나 revert 명령어를 사용해 로컬 브랜치를 이전 커밋 상태로 되돌리고, 해당 상태를 다시 원격 저장소에 푸시하는 것입니다. 두 가지 방법이 있습니다:

방법 1: git revert 사용하기

git revert 명령어는 원본 커밋을 되돌리는 새로운 커밋을 생성합니다. 이 방법은 기존의 커밋 기록을 보존하면서 변경 내용을 되돌리기 때문에, 협업 환경에서 안전하게 사용할 수 있습니다.

  • GitHub Desktop을 엽니다.
  • 왼쪽 상단의 “Current Repository” 메뉴에서 되돌리고 싶은 커밋이 있는 저장소를 선택합니다.
  • 메뉴 바에서 Repository > Open in Terminal을 선택하여 터미널을 엽니다.
  • 터미널에서 git log 명령어를 입력하여 최근 커밋 목록을 확인합니다. 되돌리고자 하는 커밋의 해시 값을 기록해 둡니다.
  • 다음 명령어를 사용하여 특정 커밋을 되돌립니다:이 명령어는 선택한 커밋의 변경 사항을 되돌리는 새 커밋을 생성합니다.
  • GitHub Desktop으로 돌아가서 "Push origin" 버튼을 클릭하여 원격 저장소에 변경 사항을 푸시합니다.

    • 방법 2: git reset 사용하기 (강제 푸시)

    git reset은 브랜치를 특정 커밋으로 되돌리지만, 해당 커밋 이후의 모든 기록을 삭제합니다. 이 방법은 협업 환경에서는 주의해서 사용해야 합니다.

  • GitHub Desktop에서 Repository > Open in Terminal을 선택하여 터미널을 엽니다.
  • 다음 명령어로 로컬 브랜치를 특정 커밋으로 리셋합니다. 되돌리고자 하는 커밋의 해시 값을 사용합니다:이 명령어는 선택한 커밋 이후의 모든 변경 사항을 삭제하고, 로컬 브랜치를 특정 커밋 상태로 되돌립니다.
  • 로컬 브랜치의 상태를 원격 저장소에 강제로 푸시하려면, 다음 명령어를 사용합니다:이 명령어는 원격 저장소의 브랜치를 강제로 업데이트합니다.

    • 주의 사항

  • revert는 협업 환경에서 안전한 방법입니다. 기존 커밋 기록을 보존하면서 변경 내용을 되돌립니다.
  • reset과 force push는 협업 환경에서 주의가 필요합니다. 다른 팀원들이 브랜치의 변경 사항을 반영하지 못해 문제가 발생할 수 있습니다.

    • 위의 방법 중 상황에 맞는 방법을 선택하여 GitHub Desktop에서 커밋을 원복하세요.
