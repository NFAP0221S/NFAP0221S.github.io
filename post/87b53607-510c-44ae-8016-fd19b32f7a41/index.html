NestJS 기본 폴더 구조

타이틀 바텀... (날짜, 이름)


src/
│
├── app.module.ts           # 최상위 모듈 (Root Module)
├── main.ts                 # 애플리케이션의 진입점 (Entry Point)
│
├── common/                 # 공통 모듈 및 유틸리티
│   ├── decorators/         # 커스텀 데코레이터
│   ├── filters/            # 예외 필터 (Exception Filters)
│   ├── guards/             # 가드 (Guards)
│   ├── interceptors/       # 인터셉터 (Interceptors)
│   ├── middlewares/        # 미들웨어 (Middlewares)
│   ├── pipes/              # 파이프 (Pipes)
│   └── services/           # 공통 서비스
│
├── modules/                # 주요 기능별 모듈
│   ├── user/               # 예시: User 모듈
│   │   ├── dto/            # 데이터 전송 객체 (DTO)
│   │   ├── entities/       # 엔터티 (Entities)
│   │   ├── user.controller.ts # User 컨트롤러
│   │   ├── user.service.ts    # User 서비스
│   │   └── user.module.ts     # User 모듈
│   │
│   ├── auth/               # 예시: Auth 모듈
│   │   ├── auth.controller.ts # Auth 컨트롤러
│   │   ├── auth.service.ts    # Auth 서비스
│   │   └── auth.module.ts     # Auth 모듈
│   │
│   └── ...                 # 추가적인 모듈들
│
└── config/                 # 설정 파일들
    ├── configuration.ts    # 애플리케이션 설정
    └── ...

세부 폴더 설명

  • modules (ex: users, auth 등):
  • common/:
  • entities/:
  • dto/:
  • pipes/:
  • guards/:
  • strategies/:

    • 이 구조는 NestJS 프로젝트에서 가장 일반적으로 사용되며, 프로젝트가 커지더라도 관리가 용이하고 확장성이 높은 구조입니다.
