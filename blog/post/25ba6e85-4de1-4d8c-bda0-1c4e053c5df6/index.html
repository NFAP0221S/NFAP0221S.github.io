앵귤러 생명주기

2024-06-01 | react


앵귤러의 생명주기 순서를 자세히 살펴보겠습니다.

1. ngOnChanges

  • 설명: 입력 프로퍼티(@Input)가 변경될 때 호출됩니다.
  • 특징: 이 훅은 컴포넌트가 생성될 때에도 한 번 호출됩니다. 변경된 값과 이전 값을 비교할 수 있는 SimpleChanges 객체를 통해 변경 사항을 확인할 수 있습니다.

    • 2. ngOnInit

  • 설명: 컴포넌트가 초기화될 때 한 번 호출됩니다.
  • 특징: 모든 입력 프로퍼티가 설정된 후에 호출되므로, 초기화 작업에 적합합니다. 외부 API 호출이나 초기 상태 설정에 유용합니다.

    • 3. ngDoCheck

  • 설명: 변경 감지를 위해 매번 호출됩니다.
  • 특징: 앵귤러의 기본 변경 감지 메커니즘에 의존하지 않고, 사용자 정의 변경 감지를 구현할 수 있는 기회를 제공합니다. 성능에 영향을 줄 수 있으므로 신중하게 사용해야 합니다.

    • 4. ngAfterContentInit

  • 설명: 컴포넌트의 콘텐츠(ng-content)가 초기화된 후 호출됩니다.
  • 특징: 자식 콘텐츠를 조작해야 할 필요가 있을 때 유용합니다. 이 시점에서는 콘텐츠가 이미 삽입되어 있습니다.

    • 5. ngAfterContentChecked

  • 설명: 콘텐츠가 체크된 후 호출됩니다.
  • 특징: 자식 콘텐츠의 변경 여부를 확인할 수 있으며, 필요한 경우 추가적인 로직을 구현할 수 있습니다.

    • 6. ngAfterViewInit

  • 설명: 컴포넌트의 뷰(템플릿)가 초기화된 후 호출됩니다.
  • 특징: 자식 컴포넌트와 뷰에 접근할 수 있으며, DOM 조작이 필요할 때 유용합니다.

    • 7. ngAfterViewChecked

  • 설명: 뷰가 체크된 후 호출됩니다.
  • 특징: 뷰에 대한 변경이 필요한지 확인하고, 추가 작업을 수행할 수 있습니다.

    • 8. ngOnDestroy

  • 설명: 컴포넌트가 파괴될 때 호출됩니다.
  • 특징: 메모리 해제 및 리소스 정리를 위한 작업을 수행합니다. 이벤트 리스너나 구독을 정리할 때 사용됩니다.

    • 결론

    앵귤러의 생명주기는 각 단계에서 컴포넌트의 상태를 관리하고, 효과적으로 데이터 바인딩 및 DOM 조작을 가능하게 합니다. 생명주기 훅을 잘 활용하면 컴포넌트의 성능과 유지보수성을 높일 수 있습니다. 이 과정을 이해하고 적절히 적용하는 것이 앵귤러 개발에서 매우 중요합니다!
